Deploy playbook

The same flow that publishes this landing page.

Push to main, publish to gh-pages, notify via Airin. Reuse the pattern for any product site or docs.

  1. Push to main. Everything under airin-landing/ triggers the workflow.
  2. Build (optional). Runs npm run buildâ€”skips cleanly for static pages.
  3. Publish to GitHub Pages. Uses peaceiris/actions-gh-pages on gh-pages.
  4. Notify. Airin posts status, branch, and run URL to your chosen hook.

FAQ

Everything you need to get started fast.

How long does setup take?

Drop the Action into your workflow, pass a webhook URL, and push. Most teams are live in under 5 minutes.

Does it work with GitLab?

Yes. Airin can be called from GitLab CI jobs with the same payload. Point the webhook at your Zapier or Telegram endpoint.

What about secrets?

Only your webhook secret is required. Airin does not touch repo contents or external services beyond that endpoint.

Can we self-host the notifications?

Absolutely. Swap the webhook URL to your own service; Airin forwards the status, branch, and run link.
