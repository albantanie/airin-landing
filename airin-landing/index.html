Ready-to-ship workflow

Deploy the landing page with Airin watching every run.

This repo ships the landing page via GitHub Pages on pushes to main. Airin posts the result so you never miss a failed publish.

  1. Push to main updates everything under airin-landing/.
  2. Build step runs npm run build (skipped for static pages).
  3. Publish via peaceiris/actions-gh-pages to the gh-pages branch.
  4. Notify Airin â†’ Zapier â†’ Telegram with the run URL and status.
